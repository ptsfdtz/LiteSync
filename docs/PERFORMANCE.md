# LiteSync 性能与压测指南（P2）

## 1. 优化项

- 全量同步支持并行复制：
  - 配置项：`jobs[].strategy.max_parallel_copies`
  - 默认值：`4`
- 增量同步支持重试退避：
  - 默认最大尝试次数：`3`
  - 退避基线：`120ms`

## 2. 推荐压测方法

## 2.1 基准测试（代码内置）

运行：

```bash
go test ./internal/backup -bench BenchmarkFullSyncParallelCopies -benchmem
```

会输出不同 worker 数（1/2/4/8）下的全量同步耗时。

## 2.2 目录规模压测

建议至少覆盖：

- 小规模：`1k` 文件
- 中规模：`10k` 文件
- 大规模：`100k` 文件（规划阶段目标）

关注指标：

- 全量同步总时长
- 增量同步延迟（事件触发到完成）
- 错误率与重试次数
- CPU/内存峰值

## 3. 回归门槛建议

- 与上一版本相比：
  - 全量同步耗时退化不超过 `20%`
  - 错误率不高于基线
  - 事件同步延迟无显著上升

## 4. 调优建议

- 如果 IO 磁盘负载高：
  - 降低 `max_parallel_copies`（如 4 -> 2）
- 如果目录深且小文件多：
  - 保持 `max_parallel_copies` 在 `4~8`
- 如果占用文件多：
  - 保持默认重试策略，结合日志定位热点路径
