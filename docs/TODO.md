# LiteSync TODO

> 状态说明：以下任务均为**规划中**。  
> 任务优先级分层：`P0`（MVP 必做） / `P1`（增强） / `P2`（体验优化）。

## P0（MVP 必做）

### [x] P0-01 配置模型与任务加载

- 目标：支持从本地 YAML 读取单个备份任务并完成校验。
- Definition of Done：
  - 可加载 `version/app/jobs` 核心字段
  - 非法路径、源目标相同、字段缺失可返回明确错误
  - 启动时能输出有效配置摘要（不泄露敏感路径细节）
- 依赖：无
- 实现状态：已完成基础版本（2026-03-05）

### [x] P0-02 首次全量同步

- 目标：首次启动或无快照时执行全量复制（源 -> 目标）。
- Definition of Done：
  - 支持递归遍历目录并复制文件
  - 支持排除规则（`exclude`）生效
  - 复制结果包含统计信息（新增/更新/跳过/失败）
  - 失败项可记录并在任务结束给出汇总
- 依赖：P0-01
- 实现状态：已完成基础版本（2026-03-05）

### [x] P0-03 文件变化监听（事件驱动）

- 目标：监听源目录变更并产出标准化事件。
- Definition of Done：
  - 支持新增、修改、删除、重命名事件
  - 监听可启动/停止，重复启动幂等
  - 监听异常可上报并触发降级策略（进入周期校验）
- 依赖：P0-01
- 实现状态：已完成基础版本（2026-03-05）
- 说明：当前已完成事件标准化与错误上报；自动降级到周期校验将在 `P0-09` 联动实现。

### [x] P0-04 调度与防抖

- 目标：将事件聚合为可执行同步任务，避免抖动导致频繁 IO。
- Definition of Done：
  - 支持 `debounce_ms` 聚合窗口
  - 同一任务不会并发执行多个同步流程
  - 支持手动触发 `SyncNow`
- 依赖：P0-02, P0-03
- 实现状态：已完成基础版本（2026-03-05）
- 说明：事件聚合已接入；当前默认路径为聚合后触发增量同步，保留“增量未实现时回退全量”的兜底机制。

### [x] P0-05 增量同步执行链路

- 目标：基于事件列表执行增量同步。
- Definition of Done：
  - 新增/修改文件可同步到目标目录
  - 删除策略支持 `propagate` 与 `ignore`（`soft_delete` 可延后）
  - 同步失败可重试（基础重试与退避）
- 依赖：P0-04
- 实现状态：已完成基础版本（2026-03-05）
- 说明：`soft_delete` 当前按 `ignore` 语义处理，后续将在 P1 阶段扩展回收站实现。

### [x] P0-06 基础日志体系

- 目标：建立可排障的日志与错误分级体系。
- Definition of Done：
  - 日志级别至少包括 `debug/info/warn/error`
  - 每次同步生成 `run_id` 并贯穿日志上下文
  - 错误可区分配置错误、权限错误、IO 错误、冲突错误
- 依赖：P0-02
- 实现状态：已完成基础版本（2026-03-05）
- 说明：已实现日志文件落盘与错误码分级（`error_code`）；后续可按需要增加日志轮转策略。

### [x] P0-07 后台运行与最小打扰

- 目标：应用可最小化到托盘并在后台持续工作。
- Definition of Done：
  - 关闭主窗口后可继续后台运行（可配置）
  - 托盘可提供“立即同步/打开界面/退出”操作（规划中 UI）
  - 退出前能完成在途任务收尾或安全中断
- 依赖：P0-04, P0-06
- 实现状态：已完成基础版本（2026-03-05）
- 说明：当前提供后台命令通道（`sync/open/exit`）作为托盘操作替代入口；托盘 UI 仍可在后续界面阶段接入。

### [x] P0-08 开机自启动基础能力

- 目标：支持开机自启动开关与状态查询。
- Definition of Done：
  - Windows/macOS/Linux 至少各有一条实现路径
  - `Enable/Disable/Status` 接口行为一致
  - 权限不足时可提示可执行的排查建议
- 依赖：P0-01
- 实现状态：已完成基础版本（2026-03-05）
- 说明：Windows 使用 Run 注册表，macOS 使用 LaunchAgents，Linux 使用 autostart desktop 文件。

### [x] P0-09 周期校验（Reconcile）

- 目标：周期性校验源/目标一致性，补偿监听漏事件。
- Definition of Done：
  - 可按 `interval_minutes` 执行校验
  - 发现漂移后可触发修复同步
  - 校验过程不阻塞 UI 线程
- 依赖：P0-05
- 实现状态：已完成基础版本（2026-03-05）
- 说明：当前 Reconcile 以“全量校验同步”方式实现，后续可优化为轻量差异扫描。

### [x] P0-10 安全退出与状态落盘

- 目标：退出时不丢失关键任务状态并避免文件损坏。
- Definition of Done：
  - 收到退出信号后停止接收新事件
  - 在途任务按可中断点安全结束
  - 最后状态与错误摘要可持久化
- 依赖：P0-04, P0-06
- 实现状态：已完成基础版本（2026-03-05）
- 说明：当前退出流程为“停止 watcher -> 停止调度器 -> 保存运行状态快照”；状态文件位于 `state_dir/runtime_state.json`。

## P1（增强）

### [x] P1-01 多任务同步

- 目标：支持多个源目录到多个目标目录的任务管理。
- Definition of Done：
  - 每个任务有独立 `job_id` 与状态
  - 支持任务级启停与并发控制
  - 单任务失败不影响其他任务执行
- 依赖：P0 全部
- 实现状态：已完成基础版本（2026-03-05）
- 说明：调度器按任务维度维护独立队列与运行循环，任务失败不会阻塞其他任务。

### [x] P1-02 冲突策略可配置

- 目标：支持 `overwrite / backup_then_overwrite / skip` 策略切换。
- Definition of Done：
  - 冲突检测规则清晰且有日志
  - `backup_then_overwrite` 能保留冲突副本
  - 策略变更可通过配置生效
- 依赖：P0-05
- 实现状态：已完成基础版本（2026-03-05）
- 说明：冲突副本默认保存到目标目录下 `.litesync_conflicts/<timestamp>/`。

### [x] P1-03 软删除回收站

- 目标：支持删除文件进入回收目录而非直接删除。
- Definition of Done：
  - 配置 `delete_policy=soft_delete` 可生效
  - 回收目录可配置保留天数（规划中）
  - 支持手动清理回收目录（规划中）
- 依赖：P0-05
- 实现状态：已完成基础版本（2026-03-05）
- 说明：当前软删除目录为 `target_dir/.litesync_trash/<timestamp>/`，保留策略与清理命令将在后续增强。

### [ ] P1-04 恢复与重放能力

- 目标：应用异常退出后可恢复未完成同步计划。
- Definition of Done：
  - 关键事件队列可落盘
  - 重启后能识别并重试未完成任务
  - 有恢复报告并可追踪
- 依赖：P0-10

### [ ] P1-05 观测性增强

- 目标：提供任务统计、最近错误、同步耗时分布。
- Definition of Done：
  - 暴露基础统计数据给 UI（规划中）
  - 错误支持按类型聚合
  - 可导出最近一次同步摘要
- 依赖：P0-06
